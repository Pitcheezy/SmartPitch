## 우선순위 2: MLP 정확도 향상 (Week 2~3)

> 시스템 전체 성능의 근본 병목. MLP 1pp 향상 = MDP + DQN + PitchEnv 전체 개선.

### Task 23: 데이터 확장 (3시즌 결합)
- **현재**: 2023 단일 시즌 72만 건
- **개선**: 2023 + 2024 + 2025(~7월) = 약 200만 건
- **방법**: `universal_model_trainer.py`에서 `statcast()` 3회 호출 + concat
- **고려사항**:
  - 시즌 간 규칙 변경 (pitch clock 2023~, shift ban 2023~) -> 영향 미미
  - 선수 이동/은퇴 -> batter/pitcher cluster 재계산 필요 여부 검토
  - 학습 시간 증가: ~40분 -> ~2시간 (데이터 3배)
- **예상 소요**: 2일 (데이터 수집 4시간 + 학습 6시간 + 검증)
- **예상 효과**: val_acc 57.5% -> **61~63%** (데이터 양 증가의 일반적 효과)
- **Fallback**: 2023 단일 시즌 유지

### Task 24: Calibration 개선
- **현재**: MLP의 `predict_proba()` 출력이 미보정 -> MDP 정책 왜곡
- **개선 방안** (구현 난이도 순):

  **방안 A: Temperature Scaling (권장, 가장 쉬움)**
  - 학습 후 검증 셋에서 최적 temperature T 탐색
  - `softmax(logits / T)` 적용 -> 확률 분포 보정
  - 수정: `model.py`의 `predict_proba()`에 T 파라미터 추가
  - 예상 소요: 0.5일
  - 예상 효과: Brier score/ECE 개선, MDP 정책 품질 향상

  **방안 B: Focal Loss (중간 난이도)**
  - `CrossEntropyLoss` -> `FocalLoss(alpha, gamma=2.0)`
  - 쉬운 샘플(ball) 가중치 감소, 어려운 샘플(foul) 가중치 증가
  - 예상 소요: 1일
  - 예상 효과: foul/hit_into_play recall 추가 개선

  **방안 C: 희귀 구종 Oversampling**
  - Knuckleball, Splitter 등 소수 구종 2~5배 복제
  - SMOTE 또는 단순 복제
  - 예상 소요: 1일
  - 예상 효과: 희귀 구종 예측 정확도 향상

- **권장 순서**: A -> B -> C (비용 대비 효과 순)
- **Fallback**: 현재 class-weighted CrossEntropyLoss 유지 (Exp5)

### Task 25: 추가 피처 도입
- **현재 사용**: 수치 6 + 물리 3 (release_speed, pfx_x, pfx_z) + 원핫 34 = 43차원
- **추가 후보**:

  | 피처 | 정규화 | 기대 효과 | 난이도 |
  |------|--------|----------|--------|
  | `release_pos_x` | (x + 2.0) / 2.0 | 팔 각도 변동 포착 | 쉬움 |
  | `release_pos_z` | (z - 6.0) / 1.0 | 릴리스 높이 변동 | 쉬움 |
  | `spin_rate` | (rate - 2200) / 500 | 무브먼트 예측 핵심 | 쉬움 (NaN ~0.5% 처리 필요) |
  | `spin_axis` | axis / 360 | 공 궤적 방향 결정 | 보통 |

- **예상 소요**: 2일
- **예상 효과**: val_acc 추가 +1~2pp
- **수정 파일**: `universal_model_trainer.py`, `model.py`, `pitch_env.py`, `mdp_solver.py`, `physical_feature_lookup.csv` 재생성
- **Fallback**: 현재 3개 물리 피처 유지

## 우선순위 4: DQN 고도화 (Week 4~5)

### Task 28: DQN 학습 스텝 증가
- **현재**: 300K timesteps
- **개선**: 500K ~ 1M timesteps
- **이유**: 상태당 평균 32.6회 방문은 부족. 군집 1~3에서 MDP 열위.
- **예상 소요**: 학습 1일 (자동)
- **예상 효과**: 군집 1~3 DQN 보상 +0.02~0.05
- **Fallback**: 300K 유지 (현재도 양의 보상 달성)

### Task 29: 탐색 전략 개선
- **현재**: epsilon-greedy (30% 탐색, 최종 eps 0.05)
- **개선 방안**:

  **A. 엔트로피 보너스 (SAC-style)**
  - 보상에 정책 엔트로피 항 추가: `reward += alpha * H(pi)`
  - Cease 같은 구종 편중 방지
  - 구현: PitchEnv의 reward 계산에 엔트로피 항 추가

  **B. Noisy Networks**
  - epsilon-greedy 대신 학습 가능한 노이즈 레이어
  - SB3 지원 여부 확인 필요

  **C. PPO 전환**
  - DQN -> PPO (정책 경사법, 엔트로피 보너스 내장)
  - SB3에서 즉시 전환 가능 (API 동일)

- **예상 소요**: 2일
- **예상 효과**: 정책 다양성 향상, Cease 유형 오버피팅 방지
- **Fallback**: 현재 epsilon-greedy 유지

### Task 30: 평가 에피소드 확대
- **현재**: 1,000 에피소드, SEM ~0.036, 95% CI 폭 ~0.14
- **개선**: 5,000 에피소드, SEM ~0.016, 95% CI 폭 ~0.063
- **이유**: 현재 에이전트 간 차이(~0.05)가 CI 폭(~0.14)보다 작아 유의성 검출 불가
- **예상 소요**: 0.5일 (평가 자동)
- **예상 효과**: 표준오차 55% 감소, 유의미한 차이 검출 가능성 증가
  - 단, Cohen's d ~0.03 검출에는 여전히 ~17,000 에피소드 필요
- **Fallback**: 1,000 에피소드 유지 (발표용으로 충분)

## 실험 원칙

1. **기존 모델 보존**: 발표용 모델(Cole/Cease/Gallen)은 절대 건드리지 않음
2. **별도 실험 환경**: 새 실험은 `experiments/` 또는 새 W&B run에서 진행
3. **A/B 비교**: 모든 변경은 기존 Exp5 대비 개선 여부를 정량적으로 검증
4. **단일 변수 변경**: 한 번에 하나만 변경하여 효과 분리
5. **재현성**: 모든 실험에 seed 고정, W&B 로깅 필수

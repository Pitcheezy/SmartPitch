## 1. 평가 지표 정리

### 1.1 MLP 전이 확률 모델

| 지표 | 수식/설명 | 현재값 | 목표 | 용도 |
|------|----------|--------|------|------|
| **val_accuracy (Top-1)** | 올바른 클래스 예측 비율 | 57.5% | 62%+ | 1차 필터 |
| **Top-2 accuracy** | 상위 2개 클래스에 정답 포함 | 80.8% | 85%+ | 모델 잠재력 |
| **macro F1** | 클래스별 F1의 산술평균 | 0.495 | 0.55+ | 소수 클래스 포함 균형 성능 |
| **Brier score** | `mean((p_i - y_i)^2)` | 미측정 | 최소화 | 확률 calibration 품질 |
| **ECE (Expected Calibration Error)** | 신뢰도-정확도 갭 가중평균 | 미측정 | < 0.05 | calibration 정밀 측정 |
| **클래스별 recall** | 각 클래스의 재현율 | ball 94%, strike 42%, foul 23%, hip 43% | 모든 클래스 > 40% | 약점 클래스 추적 |

**측정 방법**:

### 1.2 MDP (Value Iteration)

| 지표 | 설명 | 현재값 | 목표 | 용도 |
|------|------|--------|------|------|
| **VI 수렴 반복 수** | max\|DV\| < 1e-4 도달 반복 | 17회 | < 20회 | 수렴 안정성 |
| **max\|DV\|** | 마지막 반복의 최대 가치 변화 | 0.000075 | < 1e-4 | 수렴 완전성 |
| **정책 엔트로피** | 전 상태 평균 action 엔트로피 | 미측정 | > 1.0 | 정책 다양성 |
| **평균 보상** | 1000+ ep 평가 | +0.247~0.262 | > 0.28 | 정책 품질 |

### 1.3 DQN

| 지표 | 설명 | 현재값 | 목표 | 용도 |
|------|------|--------|------|------|
| **평균 보상** | n_episodes 평가 | +0.188~0.242 | > MDP | 정책 품질 |
| **표준편차** | 보상 분포 폭 | ~1.13~1.18 | 감소 추세 | 정책 안정성 |
| **SEM** | std / sqrt(n) | ~0.036 (1000 ep) | < 0.02 (5000 ep) | 추정 정밀도 |
| **95% CI** | mean ± 1.96*SEM | 폭 ~0.14 | 폭 < 0.07 | 신뢰 구간 |
| **구종 엔트로피** | 사용된 구종 분포 엔트로피 | 0.567~1.304 | > 1.0 | 오버피팅 탐지 |
| **구종 분포** | 각 구종 사용 비율 | 투수별 상이 | Statcast 기준 ±15pp 이내 | 현실성 |
| **Statcast 괴리도** | DQN vs 실제 구종 비율 최대 차이 | Cease 42pp, Gallen 13pp | < 20pp | 오버피팅 경보 |

### 1.4 통계 검정

| 지표 | 설명 | 현재값 | 판단 기준 |
|------|------|--------|----------|
| **Welch's t-test p-value** | 두 에이전트 평균 차이 유의성 | > 0.29 (모두 비유의) | p < 0.05 유의 |
| **Cohen's d** | 효과 크기 | < 0.05 (negligible) | > 0.2 small, > 0.5 medium |
| **필요 에피소드 수** | 목표 Cohen's d 검출에 필요한 n | ~17,000 (d=0.03) | 실현 가능성 평가 |

## 2. 베이스라인 정의

모든 평가에서 동일한 5개 베이스라인을 사용:

| 에이전트 | 설명 | 파라미터 |
|----------|------|----------|
| **Random** | 균등 확률로 랜덤 action 선택 | action_space 크기에 의존 |
| **MostFrequent** | 학습 데이터에서 가장 빈번한 (pitch, zone) 조합만 반복 | 투수별 데이터 필요 |
| **Frequency** | 학습 데이터의 (pitch, zone) 빈도 분포에 따라 확률적 선택 | 투수별 데이터 필요 |
| **MDPPolicy** | Value Iteration 최적 정책 | gamma=0.99, threshold=1e-4 |
| **DQN** | Stable-Baselines3 DQN | 모델별 하이퍼파라미터 |

**평가 조건 고정**:
- 에피소드 수: 1,000 (기본) / 5,000 (정밀) / 17,000 (유의성 확보)
- Seed: 0 ~ n-1 (동일 seed로 모든 에이전트 평가)
- 환경: PitchEnv (동일 MLP, 동일 action space)
- Deterministic: True (DQN, MDPPolicy)

## 3. 평가 프로토콜

### 3.1 MLP 변경 시

```
1. 새 MLP 학습 (실험 환경)
2. val_acc, macro F1, Brier score, ECE 측정
3. 기존 Exp5 대비 개선 확인
4. 개선된 MLP로 MDP 재계산 (mdp_optimal_policy.pkl 삭제 후)
5. 개선된 MLP로 DQN 재학습 (300K~ timesteps)
6. 5-agent 비교 (1000 ep)
7. 기존 베이스라인 대비 개선 확인
```

### 3.2 DQN 하이퍼파라미터 변경 시

```
1. 동일 MLP + 새 DQN 하이퍼파라미터로 학습
2. 5-agent 비교 (1000 ep, 동일 seed)
3. 기존 DQN 대비 평균 보상 + 구종 엔트로피 비교
4. 오버피팅 체크: Statcast 괴리도 < 20pp 확인
```

### 3.3 환경 파라미터 변경 시 (RE24, 인플레이 확률)

```
1. 파라미터 변경 (pitch_env.py + mdp_solver.py 동시)
2. mdp_optimal_policy.pkl 삭제
3. MDP 재계산
4. DQN 재학습
5. 5-agent 비교
6. 변경 전/후 보상 분포 비교 (분포 형태 변화 확인)
```

## 4. 오버피팅 탐지 체크리스트

모든 DQN 모델 학습 후 아래 항목 확인:

- [ ] 구종 엔트로피 > 1.0 (또는 최대 엔트로피의 70% 이상)
- [ ] 최다 구종 비율 < 60% (단일 구종 편중 경보)
- [ ] Statcast 실제 비율 대비 최대 괴리 < 20pp
- [ ] DQN 보상 >= Random 보상 (Random 이하면 정책 붕괴)
- [ ] DQN 보상 >= Frequency 보상 (학습이 데이터 빈도보다 나은지)
- [ ] 학습 보상 곡선에서 수렴 확인 (W&B 그래프)

**경보 기준**:
- 단일 구종 > 70%: **RED** (Cease 사례, 오버피팅 확실)
- 단일 구종 50~70%: **YELLOW** (주의, 투수 특성일 수 있음)
- DQN < Random: **RED** (정책 붕괴)
- 엔트로피 < 최대의 50%: **RED**

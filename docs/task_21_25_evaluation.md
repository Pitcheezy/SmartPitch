**핵심 문제**: 1루타 비율이 **45% 과소평가**되어 있었다. 실제로 인플레이 타구의 ~5개 중 1개가 1루타인데, 하드코딩은 ~7개 중 1개로 계산했다. 1루타는 주자를 한 루씩 전진시키므로, 이 괴리는 실점 기대값을 체계적으로 과소 추정하게 만들었다.

#### 순위 변동 분석

| 군집 | 이전 순위 | 현재 순위 | 변동 |
|------|----------|----------|------|
| 0 | MDP > MostFreq > Freq > Random | MDP > MostFreq > Freq > Random | 유지 |
| 1 | MDP > Freq > MostFreq > Random | MDP > Freq > Random > MostFreq | 3-4위 교체 |
| 2 | MDP > Random > Freq > MostFreq | MDP > Freq > MostFreq > Random | 2-4위 재편 |

**군집 3 역전 원인**: MDP Δ=-0.014, MostFreq Δ=-0.003. BIP 변경으로 MDP가 더 크게 영향받았다. 이전 MDP 우위가 +0.005로 미미했기 때문에 역전 발생. 다만 차이가 0.006으로 **통계적 유의성 없음** (1000 에피소드 SEM ≈ ±0.035).

### 1.2 Task 23+25: 5시즌 데이터 확장 + 확장 피처

**기대**: val_acc 57.5% → 60~62% (+3~5pp)

| 항목 | 기대 | 실측 | 판정 |
|------|------|------|------|
| 데이터 5배 확장 | 72만→350만건 | 3,551,921건 | **PASS** |
| val_acc 3~5pp 향상 | 60~62% | 57.9% (+0.4pp) | **FAIL** |
| 확장 피처 효과 | +1~2pp 추가 | +0.5pp (43→50피처) | **PARTIAL** |
| LightGBM 비교 | — | **58.7%** (MLP 대비 +0.8pp) | **신규 발견** |
| macro F1 개선 | — | 0.495→**0.509** (+0.014) | **PASS** |

#### 왜 기대(+3~5pp)에 못 미쳤는가: 갭 분석

**가설 1: 데이터 양의 한계가 아닌 피처 표현력 한계** — **입증됨**
- 5배 데이터 증가(72만→355만)에도 MLP val_acc 거의 동일 (57.5% → 57.4%)
- 학습 데이터가 부족한 것이 아니라, 현재 피처로는 투구 결과를 57~58% 이상 예측할 수 없음
- LightGBM이 18피처로 58.7% 달성 → 모델 capacity가 아닌 **피처의 예측력 자체가 한계**

**가설 2: 투구 결과의 본질적 불확실성** — **가능성 높음**
- 투구 결과는 타자의 스윙 결정, 타이밍, 심판 판정 등 관측 불가능한 변수에 크게 의존
- 투구 물리량(구속, 무브먼트, 회전)만으로는 결과를 60% 이상 맞추기 어려울 수 있음
- 이는 업계(Stuff+ 등)에서도 recognized된 한계

**가설 3: 시퀀스 정보 부재** — **미검증**
- 현재 모델은 개별 투구를 독립적으로 예측 (i.i.d. 가정)
- 실제로는 이전 투구의 위치/구종이 다음 투구 결과에 영향 (설정, 터널링 효과)
- Transformer 등 시퀀스 모델이 이 갭을 줄일 수 있으나 미검증

**결론**: 60~62% 목표는 **현재 피처셋으로는 비현실적**이었다. 57~59%가 이 피처셋의 실질적 천장이며, LightGBM이 그 천장에 가장 가까이 도달했다.

#### LightGBM의 의미

| 지표 | MLP Extended | LightGBM | 차이 | 의미 |
|------|-------------|----------|------|------|
| val_acc | 57.9% | **58.7%** | +0.8pp | 분류 정확도 |
| macro F1 | **0.509** | 0.497 | -0.012 | 소수 클래스 MLP 우위 |
| ECE | 0.0480 | **0.0025** | **-0.0455** | calibration 20배 우수 |
| Brier | 0.1392 | **0.1363** | -0.0029 | 확률 품질 LightGBM 우수 |
| 피처수 | 50 | **18** | -32 | LightGBM이 더 효율적 |

**ECE 0.0025의 의미**: 모델이 "이 투구가 strike일 확률 40%"라고 예측하면, 실제로 ~40%가 strike. MLP는 이 calibration이 4.8% 어긋남. **MDP/DQN처럼 확률 분포를 직접 사용하는 downstream에서 LightGBM이 더 정확한 의사결정을 유도할 수 있음.**

### 1.3 Task 24: Temperature Scaling

**기대**: ECE 감소 (calibration 개선)

| 항목 | 기대 | 실측 | 판정 |
|------|------|------|------|
| MLP ECE 개선 | 0.05 → 0.02 이하 | 0.0496 → **0.0714** (악화) | **FAIL** |
| MLP Brier 개선 | 감소 | 0.1411 → **0.1467** (악화) | **FAIL** |
| LightGBM | 추가 개선 | T=1.00 (불필요) | **N/A** |

**Temperature Scaling 실패 원인**:
- T=1.45 > 1.0 → 모델이 과소확신(underconfident)이 아닌 적절한 확신 수준
- Temperature를 올려 softmax를 평탄화하면, 이미 적절한 분포가 오히려 뭉개짐
- 이 데이터셋에서는 MLP의 calibration 문제가 confidence 수준이 아닌 **클래스별 bias**에서 발생

## 2. 핵심 인사이트 3가지

### 인사이트 1: "더 많은 데이터 ≠ 더 나은 모델" 실증

5배 데이터 확장이 MLP 성능을 거의 개선하지 못했다. 이는 현재 피처셋의 예측력 천장이 ~58%라는 것을 실증적으로 보여준다. **다음 돌파구는 더 많은 데이터가 아니라 더 나은 피처** (시퀀스 컨텍스트, 타자 최근 성적, 투수 피로도 등)에 있다.

### 인사이트 2: LightGBM의 calibration 우위는 RL 파이프라인에 직접 영향

MDP와 DQN은 `predict_proba()` 출력을 직접 사용해 기대 보상을 계산한다. ECE 0.0025 vs 0.048이면, LightGBM 기반 MDP/DQN이 더 정확한 가치 추정을 할 수 있다. **LightGBM으로 전이 모델을 교체하면 MDP/DQN 성능이 개선될 가능성이 높다.**

### 인사이트 3: BIP 괴리가 시뮬레이션의 가장 큰 왜곡이었다

1루타 비율 45% 과소 추정은 RE24 Δ+0.040보다 훨씬 큰 구조적 왜곡이었다. 교체 후 보상이 전반적으로 하락한 것은 **이전 시뮬레이션이 투수에게 체계적으로 유리했음**을 의미한다. 현재 보상 수치가 현실에 더 가깝다.

## 3. 발표 자료 영향도 분석

### 3.1 수정 필요한 부분

| 항목 | 이전 | 현재 | 수정 필요 |
|------|------|------|----------|
| 베이스라인 보상 표 | 2024 RE24 + 하드코딩 BIP | 2024 RE24 + 실측 BIP | **필수** |
| MLP val_acc | 57.5% (Exp5) | 57.9% (Extended) / 58.7% (LGBM) | **필수** |
| 모델 비교 | MLP 단일 | MLP vs LightGBM | **추가** |
| MDP 순위 | 전 군집 1위 | 군집 3에서 MostFreq에 역전 | **수정** |

### 3.2 새로운 어필 포인트

1. **"5시즌 355만건 통합 학습"** — 데이터 규모 자체가 어필
2. **"3개 모델 ablation study"** — 체계적 비교 방법론 강조
3. **"LightGBM ECE 0.0025"** — 확률 예측 정확도, 실전 배포 가능성
4. **"BIP 실측 데이터 교체"** — 시뮬레이션 신뢰성 근거
5. **"데이터 양이 아닌 피처가 병목"** — 실증적 발견, 연구적 가치

### 3.3 약점 (정직하게 인정할 부분)

1. **DQN이 MDP를 못 이기는 문제** — 해결 안 됨, 근본 원인은 MLP 58% 한계
2. **val_acc 천장 ~58%** — 투구 결과의 본질적 불확실성으로 제한적
3. **좌/우 분리 미적용** — platoon 피처를 추가했지만 별도 모델은 미구축
4. **군집 3 MDP 역전** — 통계적으로 무의미하나, "전 군집 MDP 1위" 주장 불가

## 4. 다음 단계 제안

### 4.1 즉시 (1~2일) — 가장 임팩트 큰 작업

**LightGBM을 전이 모델로 교체하여 MDP/DQN 재학습**
- 근거: ECE 0.0025 → MDP/DQN의 가치 추정 정확도 직접 개선
- 방법: `model.py`에 LightGBM 래퍼 추가 (`predict_proba()` 인터페이스 유지)
- 예상 효과: MDP 보상 개선 + DQN이 MDP에 더 가까워질 가능성
- 리스크: LightGBM 피처 구성이 MLP와 다름 (18 vs 43피처) → PitchEnv/MDPSolver 피처 구성 수정 필요

### 4.2 단기 (1주)

1. **DQN 학습 스텝 증가** (300K → 1M) — 현재 탐색 부족 가능성
2. **5,000 에피소드 평가** — SEM ±0.016으로 통계 유의성 확보
3. **군집별 BIP 세분화** (Task 22) — pitcher_cluster별 BIP 테이블

### 4.3 중기 (2~4주)

1. **시퀀스 모델 도입** — 이전 투구 컨텍스트 반영 (LSTM/Transformer)
2. **타자 최근 성적 피처** — 최근 7일 wOBA, 구종별 whiff rate
3. **투수 피로도 피처** — 당일 투구 수, 최근 등판 간격
